本文档说明了如何为 AIOS 提出新功能。

## 快速想法

对于快速想法或小的改进：

1. 在"想法"类别中打开讨论
2. 描述问题和建议的解决方案
3. 社区和维护者将讨论
4. 如果有兴趣，可能会升级为 RFC

## RFC 流程（用于重大功能）

对于需要设计决策的较大功能：

### 何时编写 RFC

- 新的主要功能
- 破坏性更改
- 重大架构更改
- 影响许多用户的更改

### RFC 生命周期

1. **草稿**：作者使用我们的 [RFC 模板](../../.github/RFC_TEMPLATE.md) 编写 RFC
2. **审查中**：2 周评论期
3. **决策**：维护者接受/拒绝
4. **实施**：如果被接受，实施开始

### 接受标准

- 符合项目愿景
- 技术上可行
- 有明确的实施路径
- 社区需求已证明
- 长期可维护

## 投票

- 使用 :+1: 反应表示支持
- 最受欢迎的想法优先考虑
- 维护者有最终决定权

## 时间线

- 想法：没有固定的时间表
- RFC：最少 2 周审查期
- 实施：基于路线图容量

## 从想法到实施

```text
社区想法（讨论）
        │
        │ [由维护者批准]
        ▼
内部待办项目
        │
        │ [由产品所有者优先考虑]
        ▼
冲刺规划
        │
        │ [由开发团队实施]
        ▼
发布（在 CHANGELOG.md 中标记）
```

### 谁可以添加到待办事项？

产品所有者（@po）是向内部待办事项添加项目的唯一权限。

### 贡献者鸣谢

实施其想法的贡献者将在以下地方获得鸣谢：

- CHANGELOG.md 发布说明
- 实施该功能的 PR
- 我们的贡献者页面（如适用）
