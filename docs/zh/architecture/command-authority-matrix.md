# 命令权限矩阵

此矩阵定义了 AIOS 代理之间的主要命令所有权，以避免重叠和冲突执行。

| 能力 | 主要代理 | 委托说明 |
|---|---|---|
| 故事起草/完善 | `@sm` | `@po` 委托故事创建和冲刺仪式编排 |
| 待办事项优先级 | `@po` | `@sm` 和 `@pm` 可以提议变更 |
| PRD 和战略 | `@pm` | `@po` 消费并运营化 |
| 架构设计 | `@architect` | 将数据深度工作委托给 `@data-engineer` |
| 实现 | `@dev` | 接收已验证的故事/规格 |
| 质量门禁和审查 | `@qa` | 咨询性门禁决策，向 `@dev` 提出修复请求 |
| Git 推送 / PR / 发布 | `@devops` | 远程 git 操作的独占权限 |
| 跨代理编排 | `@aios-master` | 路线修正的升级点 |

## 升级规则

如果命令所有权模糊，在执行远程影响操作之前升级到 `@aios-master`。
